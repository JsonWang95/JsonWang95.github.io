Setting Up A Personal Blog With Hexo

As an aspiring software engineer, I always think it’s crucial to have my own personal website to showcase what I’ve done.
However, everytime when I sat down and tried to start building it, I got distracted thinking about too many features I’d like to have for my website.
After reading a book called “Atomic Habit”, which I believe a lot of people have heard about, I decided to just start with a smaller goal. I came accross a
few tutorials about building a blog with a simple tool called “Hexo”, and I thought it was a good place to start. It’s really simple to setup, and I’ll
just cover a few problems that you might run into, and also some side notes that I think is good to know.

1.
When you’re deploying the website to github, you’re gonna have to modify the “_config.yml” file that’s in the root directory.
You’ll have to specify, “type”, “repo”, and “branch”. In the tutorials I found, they told us to specify the “branch” to be “master”. This might give you
some problems because GitHub actually changed the default branch to be “main”. So if you set the “branch” to be master, you’re gonna find out that your hexo files
are not being pushed to the main branch, and thus you’re actually not gonna see what you expect on your website. Now, I’m not going to talk too much about branches because it’s a big topic, but essentially there isn’t that much of a difference between main and master branch, at least for us.

TL;DR
Just make sure to set “branch” to be main, and you should be good to go.

  1. In the tutorials, they all mention this command “hexo clean”. The only explanation they give is that it’s gonna clean up all the cache and the static pages that
    were generated. That didn’t make any sense to me. But basically, it’s a command that can reset and clean up your development environment. Ok but what does this mean still? Well think of it this way. Most people use Hexo to build a blog page right? So during the process, everytime time you open a file you’re going to create some other files under the hood. And say you decide to delete some older posts, the related files might still be there hanging around. This is where “hexo clean” come into handy. You can think of it as a good practice when you’re working with hexo projects. Run it everytime before you post something new and it will help keeping your stuff nice and clean.

Also yes, it’s safe to run that command at any given time, and it won’t accidently delete any of your posts.
